# AI 视频生成工具需求与实施计划

## 1. 项目目标

在本地电脑上实现一个 AI 视频生成工具，通过调用外部 AI 接口，完成从文本文件到成品视频的自动化生成流程。

核心目标：

- 读取本地 `txt` 文本文件。
- 使用 AI 对文本进行解读、摘要、改写和脚本生成。
- 支持语音合成与语音克隆，并保存生成的音频文件。
- 支持配置视频生成方式、画面比例、分辨率、字幕、背景音乐等参数。
- 支持单数字人、单 AI 创作视频、混合数字人与 AI 视频、数字人动画形象四种生成模式。
- 生成数字人口播、AI 视频素材或动画形象视频。
- 使用本地合成能力输出最终视频文件。
- 支持项目保存、过程预览、失败重试和后续继续编辑。

整体流程：

```text
TXT 文件读取
-> 内容解读
-> 脚本生成/编辑
-> 视频生成模式选择
-> 语音克隆/语音合成
-> 视频信息配置
-> 数字人/AI 视频素材生成
-> 音频、字幕、画面混合
-> 输出成品视频
```

## 2. 用户角色与使用场景

### 2.1 目标用户

- 短视频创作者
- 企业宣传人员
- 课程内容制作者
- 产品介绍视频制作者
- 自媒体运营人员
- 需要批量生成口播视频的用户

### 2.2 典型场景

- 将一篇文章转换成短视频口播。
- 将产品说明文本转换成数字人讲解视频。
- 将科普文案转换成 AI 画面 + 配音视频。
- 将新闻或公告转换成数字人播报视频。
- 使用自己的声音克隆生成配音。
- 批量读取多个 `txt` 文件并生成多个视频。

## 3. 核心功能范围

### 3.1 文本文件读取

功能要求：

- 支持选择本地 `txt` 文件。
- 支持拖拽导入。
- 支持常见编码识别，例如 `UTF-8`、`GBK`。
- 支持展示原始文本内容。
- 支持文本内容手动编辑。
- 支持显示字数、段落数和预计口播时长。
- 支持文本过长时自动分段。
- 支持保存编辑后的文本副本。

建议增强：

- 最近打开文件记录。
- 批量导入多个 `txt` 文件。
- 敏感词或风险词提示。
- 文本为空、编码失败、文件过大时给出明确提示。

### 3.2 内容解读与脚本生成

功能要求：

- 调用 AI 文本接口解析文本内容。
- 生成内容摘要。
- 提取主题、关键词和核心观点。
- 将原文改写为适合口播的视频脚本。
- 生成分段脚本。
- 生成视频标题、简介、字幕文案。
- 为每个脚本片段生成画面描述或视频提示词。

脚本类型：

- 保持原文口播
- 简短摘要口播
- 短视频爆款风格
- 新闻播报风格
- 知识科普风格
- 产品介绍风格
- 情感故事风格
- 课程讲解风格

建议输出结构：

```json
{
  "title": "视频标题",
  "summary": "内容摘要",
  "keywords": ["关键词1", "关键词2"],
  "estimatedDuration": 60,
  "segments": [
    {
      "index": 1,
      "voiceText": "口播内容",
      "subtitle": "字幕内容",
      "visualPrompt": "画面生成提示词",
      "duration": 8,
      "sceneType": "avatar"
    }
  ]
}
```

### 3.3 视频生成模式

系统必须支持四种生成模式。

#### 3.3.1 单数字人模式

适合：

- 新闻播报
- 课程讲解
- 企业介绍
- 产品说明
- 纯口播类视频

流程：

```text
TXT 文本
-> AI 解读/改写口播稿
-> 语音合成/语音克隆
-> 数字人口型同步
-> 添加字幕/背景/音乐
-> 输出视频
```

配置项：

- 数字人形象
- 半身、全身或头像模式
- 数字人位置
- 背景图片、背景色或固定场景
- 字幕样式
- 配音声音
- 背景音乐

输出效果：

```text
一个数字人在固定画面中完整口播内容。
```

#### 3.3.2 单 AI 创作视频模式

适合：

- 风景类视频
- 故事类视频
- 产品概念片
- 情绪短片
- 科普画面
- 无真人出镜内容

流程：

```text
TXT 文本
-> AI 解读内容
-> 生成分镜脚本
-> 生成每段视频画面提示词
-> AI 生成视频素材
-> 配音/字幕/音乐合成
-> 输出视频
```

配置项：

- 视频风格
- 画面比例
- 每段时长
- 是否生成旁白
- 是否添加字幕
- 是否添加背景音乐
- AI 视频风格，例如写实、动漫、电影感、科技感、国风等

输出效果：

```text
没有数字人，整条视频由 AI 创作画面、配音和字幕组成。
```

#### 3.3.3 混合数字人与 AI 视频模式

适合：

- 知识讲解
- 带货介绍
- 新闻评论
- 产品演示
- 教程说明
- 需要“人讲解 + 画面辅助”的视频

流程：

```text
TXT 文本
-> AI 生成口播稿和分镜
-> 语音克隆/配音
-> 生成数字人口播
-> 生成 AI 视频背景/插画/素材
-> 数字人与 AI 视频混合
-> 添加字幕/音乐/转场
-> 输出视频
```

混合方式：

- 数字人在右下角，AI 视频作为背景。
- 数字人在左侧，右侧展示 AI 视频。
- 数字人全屏讲解，部分片段切换到 AI 视频。
- 数字人透明背景叠加到 AI 视频上。
- 开头和结尾使用数字人，中间穿插 AI 视频。

配置项：

- 数字人位置
- 数字人大小
- 数字人是否透明背景
- AI 视频作为背景还是插播素材
- 插播频率
- 转场样式
- 字幕样式
- 背景音乐

输出效果：

```text
数字人口播与 AI 视频画面共同组成完整视频。
```

#### 3.3.4 数字人动画形象模式

适合：

- 卡通讲解
- IP 形象口播
- 二次元角色视频
- 儿童内容
- 虚拟主播
- 品牌吉祥物讲解

流程：

```text
TXT 文本
-> AI 生成口播稿
-> 语音合成/语音克隆
-> 选择动画数字人形象
-> 生成动画口播动作
-> 添加背景/字幕/音乐
-> 输出视频
```

配置项：

- 动画形象类型：卡通、二次元、3D、扁平插画。
- 动作风格：自然讲解、活泼、严肃、可爱。
- 表情强度。
- 肢体动作强度。
- 背景场景。
- 是否搭配 AI 视频素材。
- 字幕样式。
- 配音声音。

输出效果：

```text
动画虚拟形象根据文案进行口播和动作表演。
```

### 3.4 语音合成与语音克隆

功能要求：

- 支持普通 TTS 配音。
- 支持上传音频样本进行语音克隆。
- 支持保存克隆音色配置。
- 支持选择已有克隆音色。
- 支持分段生成音频。
- 支持整段合并音频。
- 支持试听。
- 支持重新生成指定片段。

音频参数：

- 语速
- 音量
- 音调
- 情绪
- 停顿
- 旁白风格
- 是否自动加呼吸感

输入音频格式：

- `mp3`
- `wav`
- `m4a`

输出文件：

```text
audio/voice.wav
audio/segment_001.wav
audio/segment_002.wav
voice_profiles/profile_xxx.json
```

安全要求：

- 用户必须确认拥有声音使用授权。
- 不默认将用户音频上传到非必要服务。
- 语音克隆配置中不得明文暴露敏感授权信息。

### 3.5 视频信息配置

基础配置：

- 视频标题
- 输出目录
- 视频比例：`16:9`、`9:16`、`1:1`
- 分辨率：`720p`、`1080p`、`2K`
- 帧率：`24`、`25`、`30`、`60`
- 视频总时长
- 输出格式：`mp4`
- 是否生成封面
- 是否生成字幕文件

字幕配置：

- 字幕开关
- 字体
- 字号
- 字幕颜色
- 描边颜色
- 字幕位置
- 每行最大字数
- 是否烧录进视频
- 是否单独导出 `srt`

音频配置：

- 人声音量
- 背景音乐开关
- 背景音乐文件
- 背景音乐音量
- 淡入淡出
- 自动压低背景音乐

画面配置：

- 背景颜色
- 背景图片
- 背景视频
- AI 视频风格
- 数字人位置
- 水印开关
- Logo 文件
- 转场样式

### 3.6 数字人生成

功能要求：

- 支持选择数字人形象。
- 支持真人数字人。
- 支持动画数字人。
- 支持上传自定义人物图或视频，前提是接口支持。
- 支持口型同步。
- 支持输出透明背景视频，前提是接口支持。
- 支持半身、全身、头像模式。

输出文件：

```text
video/avatar_talking.mp4
video/avatar_alpha.mov
```

### 3.7 AI 视频素材生成

功能要求：

- 根据脚本片段生成视频提示词。
- 按片段生成 AI 视频素材。
- 支持文生视频。
- 支持图生视频，前提是有参考图。
- 支持失败后降级为 AI 图片或默认背景。
- 支持每段独立重试。

输出文件：

```text
video/scene_001.mp4
video/scene_002.mp4
image/scene_001.png
```

### 3.8 视频合成与导出

建议使用本地 `ffmpeg` 完成最终合成。

功能要求：

- 视频拼接。
- 音频混音。
- 字幕烧录。
- 字幕文件导出。
- 数字人与背景视频叠加。
- 视频裁剪和缩放。
- 横竖屏适配。
- 背景音乐淡入淡出。
- 添加 Logo 或水印。
- 输出最终 `mp4`。

输出文件：

```text
output/final_video.mp4
output/final_video.srt
output/cover.png
output/project.json
```

## 4. 用户体验设计

### 4.1 主流程

建议采用步骤式流程：

```text
1. 导入文本
2. 内容解读
3. 脚本编辑
4. 选择视频模式
5. 声音配置
6. 视频配置
7. 生成素材
8. 合成导出
```

每一步提供：

- 当前输入。
- AI 生成结果。
- 参数配置。
- 预览区域。
- 上一步。
- 下一步。
- 重新生成。
- 保存结果。

### 4.2 模式选择界面

用户在脚本编辑后选择视频模式：

```text
请选择视频生成方式：

[单数字人]
适合真人口播、课程讲解、新闻播报。

[单 AI 创作视频]
适合无人物出镜的画面创作类视频。

[混合数字人与 AI 视频]
适合讲解 + 辅助画面结合的视频。

[数字人动画形象]
适合虚拟主播、卡通 IP、二次元形象口播。
```

选择模式后，后续配置项按模式动态变化，避免无关配置干扰用户。

### 4.3 建议增强体验

- 一键生成完整视频。
- 分步骤生成，方便用户检查。
- 每一步都可预览。
- 支持暂停和继续任务。
- 支持失败重试。
- 支持批量处理多个 `txt` 文件。
- 支持脚本人工修改后再生成。
- 显示预计消耗时间。
- 显示预计 API 费用。
- 显示当前进度百分比。
- 支持生成前检查清单。
- 支持平台预设：抖音、视频号、B站、小红书、YouTube Shorts。
- 支持自动生成标题、简介、话题标签。
- 支持字幕错字手动修正。
- 支持多版本生成对比。
- 支持导出纯音频、纯字幕、纯脚本。

## 8. 技术实施建议

### 8.1 推荐架构

可采用本地桌面应用或本地 Web 应用。

推荐方案：

```text
前端界面
-> 本地后端服务
-> AI 接口适配层
-> 本地任务队列
-> 文件存储
-> ffmpeg 合成
```

### 8.2 模块划分

```text
FileService          文件读取与项目目录管理
ScriptService        文本解读和脚本生成
VoiceService         TTS 与语音克隆
AvatarService        数字人视频生成
AiVideoService       AI 视频素材生成
SubtitleService      字幕生成
ComposeService       ffmpeg 合成
TaskService          任务队列与状态管理
ConfigService        本地配置管理
LogService           日志记录
```

### 8.3 AI 接口适配层

不要把业务逻辑直接绑定到某一个 AI 服务商。

建议每类能力都设计统一接口：

```text
TextModelProvider
TtsProvider
VoiceCloneProvider
AvatarProvider
AiVideoProvider
```

这样后续可以替换不同服务商，而不影响主流程。

### 8.4 本地依赖

建议依赖：

- `ffmpeg`：视频合成、音频混合、字幕烧录。
- 本地数据库或 JSON 文件：保存项目与任务状态。
- 后端任务队列：处理长时间生成任务。

## 9. 异常处理与降级策略

### 9.1 常见异常

- 文本文件读取失败。
- 文本编码识别失败。
- AI 文本接口调用失败。
- 语音合成失败。
- 语音克隆失败。
- 数字人生成失败。
- AI 视频生成失败。
- ffmpeg 合成失败。
- 网络异常。
- API Key 无效。
- 余额不足。
- 输出目录无权限。

### 9.2 降级策略

- AI 视频生成失败时，使用 AI 图片或默认背景代替。
- 某个片段生成失败时，只重试该片段。
- 数字人透明背景不可用时，使用固定背景。
- 语音克隆失败时，允许切换普通 TTS。
- 字幕烧录失败时，仍导出 `srt` 文件。
- 最终合成失败时，保留所有中间产物。

## 10. 非功能需求

### 10.1 性能

- 大文本自动分段。
- 长视频分段生成后再合成。
- 生成任务不能阻塞界面。
- 支持任务进度展示。

### 10.2 稳定性

- API 调用失败自动重试。
- 每一步输出文件都落盘。
- 生成中断后可继续。
- 保存详细日志。

### 10.3 安全

- API Key 本地安全保存。
- 明确提示语音克隆授权风险。
- 用户文件只在必要时上传。
- 支持清理缓存和生成记录。

### 10.4 兼容性

- Windows 优先。
- 后续可支持 macOS。
- 依赖 `ffmpeg`。
- 需要检测本地磁盘空间。

## 11. 开发阶段计划

### 阶段一：MVP 基础闭环

目标：完成从 `txt` 到基础视频的最短可用流程。

功能：

- 读取本地 `txt` 文件。
- AI 生成口播脚本。
- 用户编辑脚本。
- 普通 TTS 生成配音。
- 生成字幕。
- 选择固定背景或固定数字人。
- 使用 `ffmpeg` 合成视频。
- 输出 `mp4`。

验收标准：

- 用户可以选择一个 `txt` 文件并生成一个可播放的 `mp4`。
- 生成过程中有进度提示。
- 失败时有明确错误信息。
- 中间产物保存在项目目录中。

### 阶段二：四种视频模式

目标：完善视频模式选择和差异化生成流程。

功能：

- 单数字人模式。
- 单 AI 创作视频模式。
- 混合数字人与 AI 视频模式。
- 数字人动画形象模式。
- 按模式动态显示配置项。
- 每种模式至少能完成一次完整生成。

验收标准：

- 四种模式在界面上可选择。
- 不同模式触发不同生成流程。
- 每种模式均能生成可播放视频或明确提示外部接口限制。

### 阶段三：语音克隆与音色管理

目标：支持用户上传音频并生成克隆声音。

功能：

- 上传音频样本。
- 调用语音克隆接口。
- 保存音色配置。
- 使用克隆音色生成配音。
- 支持试听和重新生成。

验收标准：

- 用户可以创建一个克隆音色。
- 用户可以在项目中选择该音色生成视频配音。
- 音色授权提示清晰。

### 阶段四：AI 视频素材与分镜生成

目标：提升画面自动生成能力。

功能：

- AI 生成分镜。
- 每段生成视觉提示词。
- 调用 AI 视频接口生成片段素材。
- 支持失败降级为图片或默认背景。
- 支持片段级重试。

验收标准：

- 系统能按脚本片段生成多个画面素材。
- 素材可参与最终合成。
- 单个片段失败不影响整个项目继续处理。

### 阶段五：项目管理与批量处理

目标：提高实际生产效率。

功能：

- 历史项目列表。
- 打开已有项目。
- 保存项目配置。
- 批量导入 `txt` 文件。
- 批量生成视频。
- 任务队列。
- 成本估算。
- 生成日志查看。

验收标准：

- 用户可以恢复历史项目。
- 批量任务可以顺序执行。
- 每个任务都有状态、日志和输出目录。

## 12. 后续实施优先级

优先级从高到低：

1. 项目目录与配置文件结构。
2. 文本读取与脚本生成。
3. TTS 配音生成。
4. 字幕生成。
5. `ffmpeg` 基础合成。
6. 视频模式选择。
7. 数字人接口接入。
8. AI 视频接口接入。
9. 语音克隆接口接入。
10. 项目管理与批量生成。

## 13. 待确认问题

实施前建议确认：

- 使用桌面应用还是本地 Web 应用。
- 优先接入哪些 AI 服务商。
- 是否已有数字人、语音克隆、AI 视频接口账号。
- 第一版是否必须支持语音克隆。
- 第一版是否必须支持 AI 视频生成。
- 输出视频优先面向横屏还是竖屏。
- 是否需要批量生成作为第一版功能。
- 是否需要登录、账号或本地单机使用即可。

## 14. 第一版建议范围

第一版建议控制范围，先完成可用闭环：

```text
读取 txt
-> AI 生成口播稿
-> 用户编辑
-> TTS 配音
-> 选择视频模式
-> 使用固定背景/固定数字人
-> 生成字幕
-> ffmpeg 合成
-> 导出 mp4
```

第一版暂缓：

- 复杂语音克隆。
- 高级数字人形象管理。
- AI 视频批量生成。
- 批量任务。
- 多平台自动发布。

这样可以先跑通最核心链路，再逐步扩展到完整能力。
