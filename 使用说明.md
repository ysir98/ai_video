# AI 视频生成工具使用说明

## 1. 启动程序

打开 PowerShell，进入项目目录：

```powershell
cd D:\linshi\video
```

检查运行环境：

```powershell
.\scripts\check-env.ps1
```

启动本地服务：

```powershell
.\run.ps1
```

看到类似输出表示启动成功：

```text
AI 视频生成工具已启动：http://127.0.0.1:8765/
按 Ctrl+C 停止服务。
```

浏览器打开：

```text
http://127.0.0.1:8765/
```

如果需要指定端口：

```powershell
.\run.ps1 -Port 8788
```

浏览器打开：

```text
http://127.0.0.1:8788/
```

停止服务：

```text
Ctrl + C
```

如果页面没有更新，浏览器按：

```text
Ctrl + F5
```

## 5. 导入文本

可操作项：

```text
选择 TXT
文本输入框
解读/润色脚本
清空
```

使用方式：

1. 点击“选择 TXT”导入本地 `.txt` 文件。
2. 或直接在文本框中粘贴文字。
3. 页面会显示字数和预计口播时长。
4. 点击“解读/润色脚本”生成脚本草稿。
5. 点击“清空”会清除当前文本。

注意：

```text
现在必须先生成并确认脚本草稿，才能继续“开始生成”。
```

## 6. 脚本确认

可操作项：

```text
脚本草稿编辑框
格式化 JSON
清除草稿
```

流程：

1. 在“导入文本”区域点击“解读/润色脚本”。
2. 系统会调用 AI 文本接口；如果未配置接口，则使用本地规则生成脚本。
3. 生成结果会显示在“脚本确认”区域。
4. 用户可以人工修改脚本内容。
5. 修改后可以点击“格式化 JSON”检查格式。
6. 确认无误后再点击页面顶部“开始生成”。

脚本结构示例：

```json
{
  "title": "视频标题",
  "summary": "内容摘要",
  "keywords": [],
  "style": "知识口播",
  "estimatedDuration": 60,
  "segments": [
    {
      "index": 1,
      "voiceText": "口播内容",
      "subtitle": "字幕内容",
      "visualPrompt": "画面提示词",
      "duration": 8,
      "sceneType": "avatar"
    }
  ]
}
```

重要：

- `voiceText` 用于配音。
- `subtitle` 用于字幕。
- `visualPrompt` 用于后续 AI 视频或素材生成。
- `duration` 用于字幕时间轴和渲染计划。
- 如果 JSON 格式错误，无法继续生成。

## 7. 视频生成模式

可选择 4 种模式：

```text
单数字人
单 AI 创作视频
混合数字人与 AI 视频
数字人动画形象
```

说明：

### 单数字人

适合真人口播、课程讲解、新闻播报。

当前版本会生成数字人模式的素材计划，真实数字人视频接口尚未接入。

### 单 AI 创作视频

适合无人物出镜的 AI 画面视频。

当前版本会生成 AI 视频素材计划，真实 AI 视频生成接口尚未接入。

### 混合数字人与 AI 视频

适合“数字人口播 + AI 辅助画面”。

当前版本会生成混合模式的素材计划，真实混合视频生成尚未接入。

### 数字人动画形象

适合虚拟主播、卡通 IP、二次元形象。

当前版本会生成动画数字人素材计划，真实动画数字人接口尚未接入。

## 9. 声音配置

可操作项：

```text
声音类型
已保存音色
语速
音量
上传语音样本
音色名称
保存音色样本
```

声音类型：

```text
系统语音
克隆音色
```

### 系统语音

使用 Windows 系统语音生成测试配音。

生成文件：

```text
audio/voice.wav
audio/voice.txt
```

### 克隆音色

当前版本支持保存语音样本为本地音色档案，并在后续项目中重复选择。

操作：

1. 声音类型选择“克隆音色”。
2. 上传语音样本。
3. 输入音色名称。
4. 点击“保存音色样本”。
5. 保存后可在“已保存音色”中选择。

支持格式：

```text
wav
mp3
m4a
aac
flac
ogg
```

音色保存位置：

```text
voice_profiles/voice_yyyyMMdd_HHmmss/
  profile.json
  sample.wav
```

项目中会复制所选音色配置：

```text
projects/project_xxx/voice_profiles/profile.json
```

注意：

```text
当前版本只是保存克隆样本和音色档案，还没有接入真实语音克隆 API。
实际配音仍使用 Windows 系统语音或文本兜底。
```

## 13. 生成/重新合成视频

在“项目详情”右上角点击：

```text
生成/重新合成视频
```

作用：

```text
根据当前项目的脚本、音频、字幕和渲染计划重新合成视频。
```

如果已安装 `ffmpeg`，目标输出：

```text
projects/project_xxx/output/final.mp4
```

如果没有安装 `ffmpeg`，则不会生成 mp4，只会生成或刷新：

```text
projects/project_xxx/output/render_plan.json
```

当前你的环境缺少 `ffmpeg`，所以暂时不能输出最终 mp4。

## 14. 项目目录说明

示例：

```text
projects/project_20260623_142933/
```

结构：

```text
input/
  source.txt

script/
  script.json
  script.md

audio/
  voice.wav
  voice.txt

voice_profiles/
  profile.json

image/

video/
  asset_plan.json

output/
  subtitles.srt
  render_plan.json
  final.mp4

logs/
  task.log

project.json
```

说明：

- `source.txt`：原始输入文本。
- `script.json`：结构化脚本。
- `script.md`：可读脚本文档。
- `voice.wav`：系统语音测试音频。
- `voice.txt`：配音文本。
- `profile.json`：所选音色配置。
- `asset_plan.json`：素材生成计划。
- `subtitles.srt`：字幕文件。
- `render_plan.json`：视频合成计划。
- `final.mp4`：最终视频，当前需要 ffmpeg。
- `task.log`：任务日志。
- `project.json`：项目总配置。

## 15. 推荐完整操作流程

```text
1. 启动 .\run.ps1
2. 打开 http://127.0.0.1:8765/
3. 查看运行环境
4. 导入 txt 或粘贴文本
5. 点击“解读/润色脚本”
6. 在“脚本确认”中修改和确认脚本
7. 选择视频生成模式
8. 设置脚本风格、比例、分辨率
9. 选择系统语音或克隆音色
10. 如需克隆音色，先上传样本并保存
11. 如需 AI 解读，配置文本模型 API
12. 点击“开始生成”
13. 在项目列表选择生成的项目
14. 点击“生成/重新合成视频”
15. 到 output 目录查看 render_plan.json 或 final.mp4
```
